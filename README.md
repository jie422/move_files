# 图像分析与分类工具

## 功能
1. 在线调用千文图像识别模型
2. 分析图像相似度，将相似度80%以上的图像分组并手动选择是否合并
3. 根据图像拍摄日期或文件修改日期，按年月分类存储图像

## 依赖
- Python 3.6+
- requests
- Pillow
- numpy

## 安装依赖
```bash
pip install requests Pillow numpy
```

## 配置
1. 打开 `image_analyzer.py` 文件
2. 替换 `API_KEY` 为真实的千文API密钥
3. 确认 `API_URL` 是正确的千文图像识别API地址

## 使用方法
1. 将脚本放在包含要处理图像的目录中
2. 运行脚本：
   ```bash
   python image_analyzer.py
   ```
3. 脚本会自动处理目录中的所有图像，识别相似度高于80%的图像组，并提示用户选择是否合并
4. 处理完成后，图像会按拍摄年月分类存储在对应的文件夹中

## 注意事项
- 确保网络连接正常，以便调用千文API
- 图像文件需要是常见格式（.jpg, .jpeg, .png, .bmp, .gif）
- 脚本会使用图像的EXIF信息获取拍摄日期，如果没有EXIF信息，则使用文件修改时间
- 移动图像时，如果目标文件已存在，会跳过移动操作

## 示例
假设目录中有以下图像：
- image1.jpg (拍摄于2023年10月)
- image2.jpg (拍摄于2023年10月，与image1相似度85%)
- image3.jpg (拍摄于2024年1月)

运行脚本后：
1. 会提示发现相似图像组（image1和image2）
2. 询问是否合并这些图像
3. 处理完成后，会创建以下文件夹结构：
   ```
   2023/
     10/
       image1.jpg
       image2.jpg
   2024/
     01/
       image3.jpg
   ```
